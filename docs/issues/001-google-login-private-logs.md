# 001: Googleログインと自分専用ログ

## 背景

現状は個人の読書ログを保存する用途が中心です。保存したログは自分だけが閲覧・編集できる状態にしたいです。

## やりたいこと

- Supabase Authを導入する。
- Googleログインでサインインできるようにする。
- ログイン中のユーザーだけがアプリを利用できるようにする。
- `recall_logs` に所有者を表すユーザーIDを追加する。
- 自分のログだけを閲覧・作成・更新・削除できるようにする。

## 受け入れ条件

- Googleアカウントでログインできる。
- 未ログイン状態ではログ一覧・詳細・新規作成・復習画面にアクセスできない。
- ログイン中のユーザーは自分のログだけを見られる。
- 他ユーザーのログIDへ直接アクセスしても閲覧できない。
- ログアウトできる。

## 注意点

- MVPでは認証を入れない方針だったため、実装前に認証方針のADRを追加する。
- Supabase Authの現在推奨されるNext.js App Router構成を確認してから実装する。
- Row Level Securityとpolicy設計は、このIssueまたは別Issueで明確にする。
