# 006: テスト基盤作成

## 背景

今後の機能追加に備えて、重要な処理を安心して変更できるテスト基盤を作りたいです。

## やりたいこと

- テストランナーを選定・導入する。
- バリデーションのテストを追加する。
- Data Access層のテスト方針を決める。
- 主要Server Actionsのテストを追加する。
- Custom GPT Actions向けRoute Handlerのテストを追加する。

## 受け入れ条件

- `npm test` 相当のコマンドでテストを実行できる。
- recall log作成バリデーションの正常系・異常系がテストされている。
- Custom GPT Actions APIの認証・validation error・成功ケースがテストされている。

## 注意点

- 最初は重要フローに絞る。
- Supabase依存部分をモックするか、ローカルSupabaseを使うかを先に決める。
- 導入前にNext.js 16と利用テストツールの推奨構成を確認する。
