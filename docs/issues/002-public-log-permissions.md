# 002: 公開ログと閲覧権限

## 背景

基本は自分専用ログにしたい一方で、閲覧用ログや公開してもよい学習ログは、アクセスしている全ユーザーに見せてもよい可能性があります。

## やりたいこと

- 学習ログの公開範囲を設計する。
- 非公開ログは所有者だけが閲覧できるようにする。
- 公開ログはアクセスしているユーザーが閲覧できるようにする。
- 公開ログで見せる情報と見せない情報を決める。

## 受け入れ条件

- ログごとに公開・非公開を区別できる。
- 非公開ログは所有者以外に表示されない。
- 公開ログは所有者以外でも閲覧できる。
- 公開ログでも編集・削除・復習状態更新は所有者だけが実行できる。

## 注意点

- 「閲覧ログだけ公開」の意味を、一覧公開・詳細公開・復習状態非公開などに分解して設計する。
- Supabase RLS policyで担保する範囲とアプリ側UIで制御する範囲を分ける。
