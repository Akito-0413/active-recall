# 003: 複数ユーザー対応

## 背景

将来的には、このアプリを他のユーザーも使えるようにしたいです。

## やりたいこと

- 複数ユーザー利用を前提にしたデータ分離を実装する。
- ユーザーごとの学習ログ一覧・詳細・復習画面を成立させる。
- ユーザーごとのCustom GPT Actions APIキー管理を設計する。
- 将来的なオンボーディング画面を設計する。

## 受け入れ条件

- ユーザーAとユーザーBのログが混ざらない。
- ユーザーごとの保存API認証方針が決まっている。
- 既存の単一ユーザー運用から移行する手順が整理されている。

## 注意点

- Googleログイン導入後に実装する想定。
- 既存データの所有者移行をどう扱うか決める。
- アプリ内でOpenAI APIを直接呼ぶかどうかは別途判断する。
