# Active Recall 要件定義

## スコープ

読書から得た学習内容を保存・復習するための最小Webアプリを作ります。

学習内容はCustom GPTで作成され、アプリ側ではSupabase PostgreSQLに保存します。

## ユーザーロール

MVPでは暗黙的に1種類のみです。

- 匿名学習者: 認証なしで利用する単一ユーザー想定の体験。

将来的には次を追加できます。

- Supabase Authでログインする学習者。
- 共有テンプレートや公開コンテンツを扱う場合の管理者。

## 機能要件

- 学習ソース一覧を表示できる。
- 本を中心とした学習ソースを作成・編集・削除できる。
- 学習ソースに紐づく学習エントリーを表示できる。
- GPTが生成した要約やリコール用メモを保存できる。
- 保存した学習エントリーを編集・削除できる。
- ソースのメタデータと学習内容を分けて扱える。
- MVPではログインなしで利用できる。

## コンテンツ要件

学習ソースは次を持てるようにします。

- タイトル。
- ソース種別。
- 著者または作成者。
- 任意の説明。
- 任意の外部参照。URLやISBNなど。

学習エントリーは次を持てるようにします。

- 紐づく学習ソース。
- タイトルまたは短いラベル。
- 要約本文。
- 任意のキーポイント。
- 任意の振り返り・リコール用プロンプト。
- 作成日時と更新日時。

## 非機能要件

- UIはシンプル、ミニマル、学習に集中できるものにする。
- デスクトップとモバイルの両方で使える。
- データ保存はSupabase PostgreSQLを前提にする。
- 将来的にSupabase Authを追加できる構成にする。
- 最新のNext.js 16とSupabaseの公式ドキュメントを参照し、推奨されていない設計やAPIは採用しない。
- Next.jsの実装では、必ずインストール済みドキュメントを確認する。
- 主要フローには基本的なバリデーションとエラー表示を用意する。

## MVP制約

- 認証を実装しない。
- アプリ内AI API呼び出しを実装しない。
- 自動復習スケジュールを実装しない。
- 複数ソース種別の自動取り込みを実装しない。
- 汎用ノートアプリ化しない。

## 将来要件

- Supabase Auth。
- ユーザーごとのデータ分離。
- Custom GPTからアプリへの連携改善。
- AIによるリコール問題生成。
- AIによる回答評価。
- 間隔反復スケジューリング。
- 動画、講義、記事への対応。
- ソースやエントリー横断の検索・フィルタ。
- エクスポートとインポート。

## 受け入れ条件

MVP完了の条件は次の通りです。

- 学習者が本ベースのソースを管理できる。
- 学習者がGPT生成の学習エントリーをソース配下に保存できる。
- データモデルが将来の認証・ユーザー所有に対応できる。
- 主要画面に空状態、読み込み、エラー、成功状態がある。
- lintと本番ビルドが通る。
