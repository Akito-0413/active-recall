# ADR 0005: MVPは単一のrecall_logsテーブルから始める

## Status

Accepted。

## Context

初期設計では、学習対象を表す `LearningSource` と、そこから作られる学習単位 `LearningEntry` を分ける案がありました。

その後、MVPの中心フローが「Custom GPTとの対話から3から4個の要約ポイントを持つ学習ログを保存し、未復習ログを見返すこと」に明確化されました。

MVP段階でソースとエントリーを分離すると、実装量とUI上の概念が増えます。一方で、現時点の主要データは本タイトル、要約ポイント、気づき、復習状態で表現できます。

## Decision

MVPでは単一の `recall_logs` テーブルを主データモデルとして使用します。

`recall_logs` は本タイトル、要約ポイント、気づき、復習状態、生成元を保持します。

将来、記事・動画・講義などのソース種別や複数エントリー管理が必要になった段階で、ADRを作成したうえで `learning_sources` と `learning_entries` へ分離します。

## Consequences

- MVPの実装対象が小さくなる。
- Custom GPT Actionsから保存するデータ構造が単純になる。
- 一覧、詳細、復習画面の設計が明確になる。
- 将来の正規化や移行に備えて、`source_type` と `source` を保持する。
- ソース横断の高度な整理や複数エントリー構造は、将来フェーズまで扱わない。
